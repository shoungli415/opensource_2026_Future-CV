## 자기소개

단순히 화면을 구현하는 데 그치지 않고, 사용자가 서비스를 처음 만나는 순간부터 자연스럽고 편안하게 사용할 수 있도록 전체 경험을 설계하는 데 강점을 가지고 있습니다.  

기획 의도를 이해한 뒤 이를 직관적인 정보 구조, 완성도 높은 인터페이스, 그리고 실제로 동작하는 웹 서비스로 연결하는 작업을 좋아합니다.  
앞으로는 **프론트엔드 개발자이자 UI/UX를 이해하는 프로덕트 빌더**로 성장하여, 기술과 디자인을 함께 다룰 수 있는 개발자가 되는 것을 목표로 하고 있습니다.

## 프로젝트 경험

### 1. Sidmool 화장품 사이트 리뉴얼
- 기존 화장품 쇼핑몰 사이트의 사용자 경험과 시각적 완성도를 개선하는 리뉴얼 프로젝트 진행
- 브랜드 이미지에 맞는 깔끔하고 신뢰감 있는 UI를 설계하고, 제품 탐색부터 상세 페이지 확인, 구매 전환까지의 사용자 흐름을 재구성
- 정보가 많은 뷰티 커머스 특성을 고려하여 카테고리 구조, 제품 리스트, 상세 페이지 정보 배치 등을 더 직관적으로 정리
- 단순히 화면만 새롭게 바꾸는 것이 아니라, 사용자가 원하는 제품을 더 빠르게 찾고 편하게 탐색할 수 있도록 UX 중심으로 개선
- 디자인 시안 작업과 프론트엔드 구현을 함께 수행하며, 디자인과 개발을 연결하는 역량을 강화

### 2. 테미 로봇 키오스크 개발
- 테미(Temi) 로봇 환경에서 동작하는 키오스크형 인터페이스를 기획하고 개발한 프로젝트
- 사용자가 로봇과 상호작용하는 상황을 고려하여, 일반 웹이나 모바일 화면과는 다른 사용자 흐름과 인터페이스 방식을 설계
- 큰 버튼, 명확한 정보 구조, 직관적인 화면 전환 등 키오스크 환경에 적합한 UI/UX를 반영하여 사용 편의성 향상
- 실제 사용자 입장에서 로봇과의 거리, 시선, 조작 방식 등을 고려하며 인터랙션을 설계
- 프론트엔드 개발뿐 아니라 서비스 흐름과 사용성까지 함께 고민하며 디지털 인터페이스의 확장 가능성을 경험

### 3. GitHub 기반 UI/UX 분석 및 코드 개선 서비스 개발
- GitHub 저장소를 기반으로 웹서비스의 UI/UX 품질과 프론트엔드 코드를 분석하고 개선 방향을 제안하는 서비스 개발
- 사용자가 저장소 링크를 입력하면 화면 구조, 접근성, 사용성, 코드 품질 등을 분석하여 개선 포인트를 제공하는 방식으로 기획
- 단순 코드 리뷰가 아니라, 실제 사용자 경험 관점에서 어떤 부분이 불편한지 진단하고 더 나은 인터페이스 방향을 함께 제시하는 데 집중
- UI/UX 분석 결과와 코드 개선 제안을 연결하여, 디자인과 개발을 함께 이해하는 서비스로 차별화
- 개발 생산성뿐 아니라 서비스 완성도와 사용자 경험까지 높일 수 있는 실용적인 도구를 목표로 설계

기술과 디자인 사이를 자연스럽게 연결할 수 있는 사람이 되어, 더 좋은 경험을 더 많은 사용자에게 전달하는 것이 제 커리어의 방향입니다.
